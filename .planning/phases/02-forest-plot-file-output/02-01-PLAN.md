  <behavior>
    - draw_square(): given x position (native), normalized weight, gpar args, draws a grid::grid.rect centered at (x, 0.5 npc) with width=height proportional to weight. Must be called inside a viewport with xscale set. Test: calling inside a pdf(nullfile()) device with a viewport does not error.
    - draw_diamond(): given lb, est, ub (native coords), draws 4-point grid::grid.polygon. Test: does not error when called in viewport.
    - draw_ci_line(): given lb, ub (native), draws grid::grid.segments from lb to ub at y=0.5 npc. Test: no error.
    - draw_zebra_rect(): given colshade color, draws full-width rect with that fill. Test: no error.
    - resolve_file(): given meta3l_result with name="foo", format="png", and options(meta3l.mwd="/tmp"), returns "/tmp/foo.png". With format="pdf", returns "/tmp/foo.pdf". With name=NULL, returns fallback with "forest_plot". With missing option, uses getwd().
    - auto_dims(): given n_studies, returns list(width, height) in pixels. height = max(800L, 200L + n_studies * 80L). width = 3000L. User overrides replace defaults.
    - auto_xlim(): given measure and back-transformed yi/ci values, returns appropriate xlim. PLO/PAS -> c(0,1), SMD/MD -> symmetric around 0, RR/OR -> range with margin on exp scale.
    - auto_refline(): given measure, returns NULL for PLO/PAS, 0 for SMD/MD, 1 for RR/OR.
    - format_mlab(): given i2 list (total, between, within), returns formatted string "RE Model  |  I\u00b2 = X% (between: Y%, within: Z%)"
  </behavior>
  <action>
Create `meta3l/R/forest_helpers.R` with all functions listed above. These are internal (not exported) — no `@export` tags. Add roxygen `@keywords internal` blocks for documentation.

Key implementation notes:
- All draw_* functions assume they are called inside an active grid viewport with xscale already set. They do NOT push/pop viewports themselves.
- draw_square: `grid::grid.rect(x=unit(x_pos, "native"), y=unit(0.5, "npc"), width=unit(size, "lines"), height=unit(size, "lines"), just="centre", gp=gpar(fill=col, col=col_border))`
- draw_diamond: `grid::grid.polygon(x=unit(c(lb, est, ub, est), "native"), y=unit(c(0.5, 0.5+hh, 0.5, 0.5-hh), "npc"), gp=gpar(fill=col, col=col_border))`
- draw_ci_line: `grid::grid.segments(x0=unit(lb, "native"), x1=unit(ub, "native"), y0=unit(0.5, "npc"), y1=unit(0.5, "npc"), gp=gpar(lwd=lwd))`
- draw_zebra_rect: `grid::grid.rect(x=unit(0.5, "npc"), y=unit(0.5, "npc"), width=unit(1, "npc"), height=unit(1, "npc"), gp=gpar(fill=col, col=NA))`
- resolve_file: Use `missing()` check pattern if called from forest.meta3L; but since this is a helper, accept explicit `file` argument. If `file` is `character(0)` (sentinel for auto-name): assemble from `x$name` + format + `getOption("meta3l.mwd", default=getwd())`. If `file` is NULL: return NULL (display only). If `file` is a string: return as-is.
- auto_dims: Simple formula. Return `list(width=if(!is.null(user_w)) user_w else 3000L, height=if(!is.null(user_h)) user_h else max(800L, 200L + n * 80L))`
- auto_xlim: Back-transformed range with 5% margin padding. For PLO/PAS clamp to c(0,1).
- auto_refline: Simple switch on measure.
- format_mlab: `sprintf("RE Model  |  I\u00b2 = %.0f%% (between: %.0f%%, within: %.0f%%)", i2$total, i2$between, i2$within)`

R >= 4.0 compatibility: no `|>` or `\(x)`.

Create test file with tests for resolve_file, auto_dims, auto_xlim, auto_refline, format_mlab (pure logic, no device needed), plus smoke tests for draw_* functions (open pdf(nullfile()), push viewport, call function, no error).
